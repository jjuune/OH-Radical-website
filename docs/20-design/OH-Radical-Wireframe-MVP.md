# OH Radical — 메인 페이지 MVP 와이어프레임 구조 설계서

> **문서 성격:** 내부 공유용 / 외주 제작 청사진 / 개발·디자인 참고용  
> **기준 UX Flow:** OH Radical UX Flow Strategy v1.0  
> **뷰포트 기준:** Desktop 1440px / Mobile 390px  
> **구현 레벨:** MVP (Phase 1) — 브랜드 방향성 검증 우선, 과도한 인터랙션 배제

---

## 📐 설계 원칙 (Design Principles)

```
1. 여백이 메시지다      → 콘텐츠를 넣는 것이 아니라 덜어내는 설계
2. 구조가 스토리다      → 섹션 순서 자체가 설득의 흐름
3. 인터랙션은 도구다    → 연출이 아닌 이해를 위한 모션만 허용
4. 모바일은 독립 경험  → 데스크탑 축소가 아닌 재구성
```

## Section 0. Global Navigation

### 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────────┐
│  [LOGO]  OH Radical          Tech  Solutions  About   [문의하기] │
│  ← 좌 고정                    ← 중앙 정렬 메뉴 →      ← 우 고정 CTA │
└─────────────────────────────────────────────────────────────────┘
  높이: 72px
  배경: 초기 = transparent / 스크롤 시 = rgba(4,10,15,0.85) + blur(12px)
```

### 콘텐츠 배치

- **좌:** 로고 워드마크 (`OH Radical`) — 크기 22px, 자간 넓게
- **중:** 메뉴 4개 — `Technology` / `Solutions` / `About` / `Contact`
- **우:** CTA 버튼 `문의하기` — 아웃라인 스타일 (흰 테두리 / 투명 배경)
- **언어 전환:** CTA 버튼 좌측에 `KO | EN` 소형 토글

### Sticky 동작

```
초기 상태   →  배경 transparent, 로고 White
스크롤 72px+ →  배경 Glassmorphism (backdrop-blur), 보더 하단 선 0.5px 등장
```

### 모바일 대응

```
┌─────────────────────────┐
│  [LOGO]        [☰ 메뉴] │
└─────────────────────────┘
  → 햄버거 메뉴 → 풀스크린 오버레이 (Fullscreen Nav Overlay)
  → 오버레이: 세로 메뉴 리스트 + 하단 문의하기 버튼
```

### 정보 밀도 가이드
- 메뉴 항목 최대 4개 유지 (드롭다운 없음 — MVP 기준)
- 불필요한 아이콘 배제 — 텍스트만으로 구성

### 콘텐츠 배치 상세

```
레이어 구조 (bottom → top):

Layer 1 (배경):  Deep Space Black (#040a0f) 단색 배경
Layer 2 (파티클): Canvas 기반 미세 입자 — 밀도 낮게, 컬러 Bio-Cyan 5% opacity
Layer 3 (콘텐츠):
  - 메인 헤드라인:  "Human First." / "Earth Always."
                   폰트 크기: 96px (desktop) / 52px (mobile)
                   자간: -0.02em, 줄간: 1.05
                   위치: 화면 중앙 세로 기준 45% (약간 위)

  - 서브 헤드:     "The invisible defense system."
                   한글 보조: "보이지 않는 곳에서 작동하는 안전 인프라."
                   폰트 크기: 18px / opacity 0.65
                   헤드라인 하단 24px 간격

  - CTA 버튼:      [솔루션 탐색하기] — Outline 버튼 (흰 테두리, 투명 배경)
                   [기술 알아보기 →] — 텍스트 링크 + 화살표
                   서브헤드 하단 48px 간격, 가로 간격 24px

Layer 4 (스크롤 인디케이터):
  - 화면 하단 중앙, 얇은 세로 선(40px) + 아래 방향 애니메이션
  - 텍스트 "Scroll" 또는 없이 — 선만으로 충분
```

### 파티클 시스템 (MVP 간소화)

```
MVP 기준: CSS animation 기반 pseudo-element 점들 (JavaScript Canvas 미사용)
  → 50~80개 소형 점(1~2px) / opacity 0.03~0.08 / 느리게 부유 (20~40s loop)
  → 마우스 인터랙션 없음 (Phase 2로 이월)
  → 성능 부담 최소화
```

### 진입 애니메이션 (MVP)

```
0ms      → 화면 로드, 배경 파티클 페이드인 (opacity 0 → 1, 800ms)
400ms    → 헤드라인 1행 등장 (translateY(20px) → 0, opacity 0 → 1, 600ms)
700ms    → 헤드라인 2행 등장 (동일, stagger 300ms)
1000ms   → 서브헤드 등장
1300ms   → CTA 버튼 등장
1600ms   → 스크롤 인디케이터 등장
```

### Sticky Scroll 동작 원리

```

구간 1 (0~100vh)   → 🦠 Viruses     노출
구간 2 (100~200vh) → ⚗️ VOC          노출
구간 3 (200~300vh) → 💨 악취         노출
구간 4 (300~400vh) → 🍳 조리흄      노출
구간 5 (400~500vh) → 🌫️ 미세먼지    노출

구현 방식 (MVP):
  → Intersection Observer API 기반 (GSAP ScrollTrigger 미사용)
  → 각 구간 진입 시 className 변경으로 콘텐츠 전환
  → CSS transition으로 fade 처리 (0.4s ease)
```

### 3개 항목 구성 요약

```
Step 1: ❌ Filter-Free (무필터)
  좌: 텍스트 + 비용 절감 수치
  우: 복잡한 필터 구조 → 심플한 기류 이미지 (Before/After)

Step 2: ❌ Chemical-Free (무화학약품)
  좌: 텍스트 + "최종 산물: H₂O" 강조
  우: 약품 아이콘들 → 물 분자 아이콘으로 교체 (단순 SVG)

Step 3: ❌ Zero Waste (무폐기물)
  좌: 텍스트 + ESG 연계 메시지
  우: ESG 순환 인포그래픽 (정적 SVG)
```

### 배경 전환

```
Why It Matters 퇴장 후: 배경 점진적으로 밝아짐
3무 섹션 전체: 매우 어두운 배경에서 → 미세하게 청정한 느낌으로 전환
  → 각 Step마다 배경에 Mint/Cyan tint 미세 증가
  → Step 3 퇴장 시: 가장 맑은 상태 (다음 Technology 섹션 준비)
```

### 모바일 대응

```
→ Sticky Scroll → 탭 기반 전환 (Tab 1 / Tab 2 / Tab 3)
→ 탭 하단에 각 Step 콘텐츠 수직 나열
→ 비주얼은 텍스트 상단 배치
```

### 정보 밀도 가이드
- 각 Step: 헤드라인 1줄 + 서브 헤드 1줄 + 본문 최대 3줄 + 수치 1개
- 배지/태그는 ❌ 기호 하나로 단순화
- 아이콘 남발 금지

## 🔧 기술 구현 가이드라인 요약

```
Framework:   Next.js (App Router)
Styling:     Tailwind CSS + CSS Modules (커스텀 필요 구간)
Animation:   Framer Motion (컴포넌트 단위 진입 애니메이션)
Scroll:      Intersection Observer API (Sticky Scroll)
             → GSAP ScrollTrigger는 P2 이후 도입 검토
3D/파티클:   MVP = CSS animation 기반 pseudo-element
             P2 = Canvas API 또는 Three.js
슬라이더:   Swiper.js (케이스 카드, 산업 솔루션)
폼:         React Hook Form + 이메일 연동 (Resend or EmailJS)
아이콘:     Lucide React (통일성)
SVG 일러스트: 직접 제작 또는 외주 (기술 다이어그램 3종)
```

## 🚫 금지 사항 (Anti-Pattern Checklist)

```
레이아웃:
  ❌ 섹션 내 3개 이상의 컬럼 사용 (모바일 대응 복잡도 증가)
  ❌ 플로팅 팝업/모달 남발
  ❌ 스크롤 방해 요소 (고정 사이드바 등)

콘텐츠:
  ❌ 제품 스펙 표(Table) 배치 (카탈로그 느낌)
  ❌ 인증서 이미지 직접 노출 (배지 형태로 대체)
  ❌ 뉴스/블로그 섹션 메인 노출 (Footer 링크로 이동)
  ❌ 가격 정보 노출

비주얼:
  ❌ 장비/기기 외형 사진 중심 구성
  ❌ 총천연색 아이콘 세트 사용
  ❌ 인포그래픽 남발 (최대 2~3개 섹션)
  ❌ 배경 그라데이션 (보라/파랑 → 분홍 계열 절대 금지)

인터랙션:
  ❌ 자동 재생 슬라이더 (Hero 배너 슬라이더 금지)
  ❌ Cursor 커스터마이징 (MVP 단계 불필요)
  ❌ 페이지 전환 애니메이션 (첫 MVP에서 불필요)
```
